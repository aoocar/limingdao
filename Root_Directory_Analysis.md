# Limingdao (WebStack-Hugo) 根目录文件与文件夹功能分析

本项目是一个基于 **Hugo** 静态站点生成器的网站项目（基于 WebStack 主题的网址导航项目）。

以下是项目中一级目录下的文件和文件夹的功能与作用分析：

## 📁 核心文件夹 (Hugo 结构)

| 文件夹 | 功能说明 |
| --- | --- |
| `content/` | **内容目录**。这是 Hugo 项目的核心，存放网站的所有 Markdown 文件或 HTML 文件。网址导航的具体数据、页面内容都存放在这里。 |
| `layouts/` | **布局目录**。存放 HTML 模板文件。用于覆盖主题默认的布局文件，进行网站结构或样式的自定义。 |
| `static/` | **静态资源目录**。存放图片（Logo、Favicon）、CSS 文件、JavaScript 文件等不需要 Hugo 进一步处理的静态文件。打包时原封不动复制到根目录下。 |
| `themes/` | **主题目录**。存放下载的 Hugo 主题（如 WebStack 主题）。 |
| `data/` | **数据目录**。用于存放配置数据文件（如 YAML、JSON 或 TOML 格式）。部分导航站的分类和网址数据可能会以数据文件的形式存放在这里。 |
| `assets/` | **资产目录**。存放需要 Hugo 管道（Hugo Pipes）处理的资源文件，如 SASS/SCSS、需要压缩的 JS 或 CSS 文件等。 |
| `archetypes/` | **内容模板目录**。使用 `hugo new` 命令创建新内容时使用的 `.md` 模板（Front Matter 预设）。 |
| `i18n/` | **国际化目录**。存放多语言翻译文件（如 `en.toml`, `zh-cn.toml`），支持网站的多语言显示。 |
| `public/` | **发布目录**。运行 `hugo` 命令构建网站时生成的最终静态 HTML 文件和相关资源，用于部署到服务器。 |
| `resources/` | **资源缓存目录**。Hugo 处理过的图片、CSS 等资源会缓存在这里，以加快后续的构建速度。 |

## 📁 项目特定/工具文件夹

| 文件夹 | 功能说明 |
| --- | --- |
| `.git/` | Git 版本控制系统的内部文件夹，记录项目的历史提交和分支信息。 |
| `.github/` | GitHub Actions 工作流配置文件夹，通常包含用于自动化部署（CI/CD）的 `.yml` 配置文件。 |
| `.obsidian/` | Obsidian 知识管理软件的配置文件夹。说明项目同时被作为 Obsidian 库打开，用于项目文档管理或内容编写。 |
| `skills/` | 自定义文件夹，可能存放项目相关的技能描述、脚本或特定功能的文档/代码。 |

## 📄 核心文件

| 文件 | 功能说明 |
| --- | --- |
| `config.toml` | **Hugo 站点主配置文件**。控制网站的基础设置，如网站名称、基础 URL、默认语言、使用的主题、导航栏菜单配置等。 |
| `vercel.json` | **Vercel 部署配置文件**。配置重定向、自定义路由或其他 Vercel 相关的构建选项，说明项目可能部署在 Vercel 上。 |
| `.gitignore` | Git 忽略配置文件。指定不需要被版本控制和上传到代码仓库的文件或文件夹。 |
| `.hugo_build.lock`| Hugo 构建时的锁文件，防止多个 Hugo 进程同时构建导致冲突。 |
